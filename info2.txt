# Professional Prompt: Fix ANSI Color Rendering in Ferox Framework on Windows

## Context
I have a Rust-based penetration testing framework called **Ferox** that successfully compiles and runs on Windows. However, when executed in PowerShell or CMD, the ANSI escape codes for colors and Unicode symbols render incorrectly, displaying raw escape sequences like `[0m`, `[91m`, `[93m` instead of colored output.
