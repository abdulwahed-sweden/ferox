Reconnaissance Phase

01 Subdomain Enumeration

Basic

This scenario enables comprehensive subdomain information gathering using multiple data sources including DNS records, certificate transparency logs, and search engines.

Important Note

Ensure you have proper authorization before conducting any reconnaissance activities on target systems.

ferox-console
ferox> use recon/subdomain_enum ferox> set DOMAIN example.com ferox> set THREADS 50 ferox> set OUTPUT ./results/subdomains.txt ferox> run # Expected output: # [+] Found: api.example.com # [+] Found: mail.example.com # [+] Found: admin.example.com

Advanced Options

Parameter Description Default Value
DOMAIN Target domain Required
THREADS Number of concurrent threads 10
WORDLIST Custom wordlist path default.txt
RECURSIVE Enable recursive search false

Security Warning

Using a high number of threads may result in your IP being blocked. Start with lower values and increase gradually.

02 DNS Record Analysis

Advanced

Comprehensive DNS record analysis to discover infrastructure details and associated services of the target organization.

ferox-console
ferox> use recon/dns_analysis ferox> set TARGET example.com ferox> set RECORD_TYPES A,AAAA,MX,TXT,NS,SOA ferox> set DNS_SERVER 8.8.8.8 ferox> run

Pro Tip

Use multiple DNS servers to get more comprehensive results and verify consistency across different resolvers.

Scanning Phase

04 Advanced Port Scanning

Basic

Perform comprehensive port scanning using multiple techniques to evade detection while maintaining accuracy.

ferox-console
ferox> use scan/port_scanner ferox> set TARGET 192.168.1.0/24 ferox> set PORTS 1-65535 ferox> set SCAN_TYPE SYN ferox> set RATE 1000 ferox> set TIMEOUT 3000 ferox> run

Supported Scan Types

  • SYN Scan: Half-open scan, fast and stealthy
  • TCP Connect: Full connection scan, more reliable
  • UDP Scan: For detecting UDP services
  • ACK Scan: For firewall rule detection
  • FIN/NULL/Xmas: For evading some detection systems

05 Automated Vulnerability Detection

Advanced

Automated scanning for known vulnerabilities using the latest CVE database and custom detection signatures.

ferox-console
ferox> use scan/vuln_scanner ferox> set TARGET 192.168.1.100 ferox> set SCAN_TYPE comprehensive ferox> set CVE_DB latest ferox> run # Sample output: # [CRITICAL] CVE-2021-44228 - Log4Shell # [HIGH] CVE-2021-34527 - PrintNightmare # [MEDIUM] CVE-2020-1472 - Zerologon

Exploitation Phase

06 Web Application Exploitation

Expert

Comprehensive web application testing toolkit including SQL Injection, XSS, and CSRF testing capabilities.

Important Warning

Only use these tools on systems you have explicit authorization to test. Unauthorized use is a criminal offense.

ferox-console
# SQL Injection Testing ferox> use exploit/web/sqli ferox> set URL "http://target.com/login.php" ferox> set PARAM username ferox> set TECHNIQUE UNION ferox> set DBMS MySQL ferox> run

Supported SQL Injection Techniques

Technique Description Effectiveness
UNION Extract data via UNION SELECT High
Boolean-based Infer data from response differences Medium
Time-based Infer data from time delays Slow
Error-based Extract data from error messages High

07 Password Cracking

Advanced

Advanced tools for auditing password strength and testing authentication mechanisms across multiple protocols.

ferox-console
ferox> use exploit/auth/brute_force ferox> set TARGET ssh://192.168.1.100 ferox> set USERLIST ./wordlists/users.txt ferox> set PASSLIST ./wordlists/rockyou.txt ferox> set THREADS 10 ferox> set DELAY 500 ferox> run

Supported Protocols

SSH, FTP, Telnet, HTTP Basic/Digest, SMB, RDP, MySQL, PostgreSQL, MSSQL, Oracle, VNC, LDAP
